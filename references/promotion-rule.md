# Promotion Rule

Use this rule when deciding whether a change belongs in the canonical kit or should stay local to a consuming app.

Promote a change back into the kit when it is:

- foundational rather than app-specific
- likely to be reused in the next project
- part of the token, theme, merge-policy, typography, or button contract
- necessary to keep cross-file semantics aligned

Keep a change local when it is:

- branding for a single app
- a temporary experiment
- a wrapper or convenience abstraction tied to one product flow
- a control pattern that has not yet proved durable across projects

When unsure, keep it local first. Promote it only after it survives real reuse.
